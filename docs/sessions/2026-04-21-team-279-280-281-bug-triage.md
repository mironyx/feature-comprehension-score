## Post-PR cost breakdown

Three recurring costs compound across every `/feature-end`:

**1. `/lld-sync` (unavoidable, ~$1–1.5 per issue)**
Reads the full LLD file (~960 lines for E17) plus the changed source files, produces a diff, rewrites the doc. This is the dominant intrinsic cost of `/feature-end` itself. For small bug fixes that only touch implementation details already described in the LLD, the sync adds marginal value. Consider: skip lld-sync for pure bug fixes (label `bug`) where the LLD contract hasn't changed — only run it for features and design changes.

**2. Extra CI cycle from pre-existing `main` breakage (~$1–1.5 per issue, avoidable)**
Commit `92057f0` landed `bug-report-21-04-26.md` without an H1, breaking `markdownlint` for every subsequent branch. All three teammates discovered this at CI time, added a fix commit, and waited for a second CI run. Running `npx markdownlint-cli2 "**/*.md"` on `main` before spawning agents would catch this in seconds and allow a single fix commit — avoiding N×CI cycles.

**3. Rebase + second CI run (#280 and #281, ~$2–3 per issue, avoidable)**
When #279 merged first, #280 and #281 needed to rebase. The rebase itself is cheap. What's expensive is: `gh run watch` blocks and streams CI output (5–6 min per run), and the agent reads all of it. For a rebase that only touches doc files (no source changes), it should be safe to squash-merge without waiting for a second CI run — or at minimum commit the rebase with `[skip ci]` when the only change is a docs conflict resolution.

**Net avoidable cost this run: ~$12–14 out of $20 post-PR delta.**

## Process notes for `/retro`

1. **Scan main for lint violations before spawning.** Run `npx markdownlint-cli2 "**/*.md" --ignore node_modules` at `/feature-team` Step 1. Fix any breaks with a single `chore:` commit before cutting worktrees.
2. **Consider skipping lld-sync for `bug` label issues.** When the LLD contract hasn't changed (only implementation details), lld-sync adds $1–1.5 with low signal. Gate it on whether the issue is a feature vs. bug fix.
3. **Rebase-only commits should use `[skip ci]`.** When a rebase resolves only doc conflicts (no source changes), the follow-up CI run is redundant. Add `[skip ci]` to the force-push commit message in the rebase step of `/feature-end`.
4. **Export `PROM_PORT` in `.env.test.local`.** Worktrees don't inherit shell env vars set only in `.bashrc`. Cost queries fail silently when `PROM_PORT` defaults to 9090.
5. **Teammates must report completion after `/feature-end`.** Skill prompt says "Report back… and wait" at PR time but not after feature-end. Add an explicit step.
