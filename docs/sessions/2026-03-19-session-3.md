## Work Completed

### External review feedback addressed

An automated review (devloai) identified two correctness bugs, both fixed:

1. **Service role client used wrong library** (`createServerClient` from `@supabase/ssr` → `createClient`
   from `@supabase/supabase-js`). The SSR client inspects request cookies and can override the
   `Authorization` header with a user JWT, silently downgrading service role access. Fixed in commit
   `68c8403`.

2. **RSC `setAll` missing try-catch** — `cookieStore.set()` throws inside React Server Components.
   The Supabase SSR docs require a silent catch here; middleware handles the refresh on the next
   request. Fixed in commit `d31d78d`.

### Env var extraction — `env.ts` shared module

External review (and cross-session feedback) identified verbatim copy-paste of env var validation
across four files. Extracted to `src/lib/supabase/env.ts` using a `?? IIFE` pattern:
- TypeScript infers `string` without type assertions
- `serviceRoleKey` intentionally not exported — only consumed in `service-role.ts`

### CI failure — test env var fallbacks

Module-level env var validation in `env.ts` fires at import time. CI has no Supabase env vars.
Fixed by adding `??=` fallbacks in `tests/setup.ts` before any module import.

### Test assertion hardened

The service role key assertion was circular (`expect(keyArg).toBe(process.env['SUPABASE_SERVICE_ROLE_KEY'])`).
Fixed to use the fixed literal `'test-service-role-key'` set by `tests/setup.ts`.

### Grafana dashboard — `max_over_time` fix

`claude_code_cost_usage_USD_total` is a gauge (not a counter) that resets to 0 on each new
Claude Code process start. Plain instant vector queries only captured the current session's
running total, showing incorrect (understated) totals.

Fix: `max_over_time(claude_code_cost_usage_USD_total[$__range])` captures peak value per session.
Cross-validated against `increase([7d])` — both return $1.38. `deriv([10m]) = 0` confirms the
gauge is settled.

Same fix applied to token total panel. Token time-series changed from raw gauge to `increase([5m])`
for consistency with the cost time-series panel.

Updated: `monitoring/grafana/dashboards/fcs-feature-cost.json` (version 1 → 2).
