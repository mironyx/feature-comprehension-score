## Decisions made

- **Two enforcement modes: Soft (default) and Hard** — Soft tests engagement (relevance only); Hard tests understanding (relevance + score threshold). Semantically distinct, not just quantitatively different. Documented in ADR-0006.
- **Scoring always runs in both modes** — In Soft mode, scores are calculated and stored for reporting but do not affect gate outcome. Accepted V1 trade-off: calibration data justifies the LLM cost.
- **"Off" mode rejected** — PRCC's enabled/disabled flag already covers the zero-consequence case. A third mode adds confusion without benefit.
