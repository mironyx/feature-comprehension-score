## Decisions made

- **`SignInButton.tsx` split from `page.tsx`** ‚Äî Next.js App Router requires `'use client'` for interactive elements; the server component handles the `?error` query param (a Promise in Next.js 15 that must be `await`ed server-side).
- **`storeProviderToken` not extracted as helper** ‚Äî single call site; no reuse benefit. Inlined in the GET handler.
- **`/` is protected, not public** ‚Äî spec listed `/` as public. Correct behaviour is to protect it ‚Äî unauthenticated requests redirect to sign-in. Matcher pattern excludes only `auth/` and `api/webhooks/` prefixes.
- **Org membership sync deferred** ‚Äî callback spec included org sync as step 5. Deferred to ¬ß2.3 / future issue. Redirect hardcoded to `/assessments` (no `returnTo` support yet).
- **`output: standalone` + `next start` incompatibility** ‚Äî `next start` does not work with standalone output. `npm run start` changed to `node .next/standalone/server.js`. E2E tests now rely on `.env.local` for local env vars rather than inline env block in playwright config.
- **ci-probe `permissionMode: bypassPermissions`** ‚Äî project-level `permissions.allow` in `settings.json` does not apply to background subagents. The `permissionMode` frontmatter field in the agent definition is the correct mechanism.
