## Decisions made

- **Decompose E11.3 into 2 tasks** (T3.1 settings page + glob validation; T3.2 resolver + rubric wiring) — clean FE/BE seam, ~580 lines combined exceeds the 200-line PR bar.
- **Schema extension over parallel schema** (Option A): extend `OrganisationContextSchema` with optional `glob_patterns` and `question_count` rather than introducing `ProjectPromptContextSchema`. Org-level rows are FCS-inert in V11; the extra optional fields cost nothing.
- **Raise `domain_notes` cap** from 500 to 2000 to match `UpdateProjectSchema` (already accepts up to 2000).
- **Retain `loadOrgPromptContext`** — still used by the org-context UI on `/organisation`. Only the FCS path's call is removed.
- **No new ADR.** Decisions are covered by ADR-0027/0028 and the V11 HLD §C4 amendment to ADR-0013.
- **Glob validator: `picomatch`** (added as direct dep). Lightweight; transitively available already.

## Refinement vs. plan

The plan's parallelisation map ([§Parallelisation Map](../plans/2026-04-30-v11-implementation-plan.md#parallelisation-map)) marks **E11.2 ↔ E11.3 as `parallel-safe`** based on disjoint Owns. File-level analysis refines this:

- **T3.1 ↔ all of E11.2** — parallel-safe.
- **T3.2 ↔ E11.2 T2.2 (#411)** — **conflicting** (both modify `extractArtefacts`).
- **T3.2 ↔ all other E11.2 tasks** — parallel-safe.

Recommended plan patch: change the E11.2 ↔ E11.3 edge to `coordinate rubric pipeline ownership — T3.2 follows T2.2`.
