## Decisions made

- **Pino over Winston/Bunyan** — smaller, faster, JSON-native, built-in error serialisation (ADR-0016)
- **`{ err: error }` convention** — Pino's built-in error serialiser requires the `err` key as first arg
- **Client components keep `console.error`** — Pino is Node-only; browser logging stays as-is
- **Contextual logging (child loggers per module) deferred** — tracked as follow-up #145
- **docs/INDEX.md for documentation navigation** — tracked as follow-up #146
