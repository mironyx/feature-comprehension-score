## What was built

**New files:**

**Key design choices:**

- Fire-and-forget rubric generation: `void triggerRubricGeneration(...)` — assessment creation returns 201 immediately
- Branded ID types (`OrgId`, `UserId`, `RepositoryId`, `AssessmentId`) — eliminates Primitive Obsession; zero runtime overhead; casts at the service boundary only
- `FcsCreateInput = Pick<FcsCreateBody, ...>` — narrowed alias enforcing the LLD §2.4 constraint that only a subset of body fields reach the DB write
- `createAssessmentRecord` generates its own UUID and returns `AssessmentId` (4 params, not 5 as specified) — required to satisfy CodeScene's excess-arguments threshold

**Skill updates:**

- `.claude/skills/diag/SKILL.md` — open-first protocol: open files in Windsurf before editing so PostToolUse hook gets live diagnostics after each save; replaced all `code <file>` with `windsurf --reuse-window <file>`
