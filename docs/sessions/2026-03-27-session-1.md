## Decisions made

- **Server/client split:** `page.tsx` is a pure server component; `answering-form.tsx` holds the `'use client'` boundary. This keeps DB access and auth checks server-side while enabling React state for the interactive form.
- **No internal API hop:** Server component fetches directly from Supabase via admin client rather than calling `GET /api/assessments/[id]`. This avoids a round-trip and is consistent with the existing results page pattern.
- **`useAnsweringForm` hook:** Extracted to keep `AnsweringForm` component body below cc=10 (CodeScene hard limit). State and handlers in the hook; JSX-only in the component.
- **`attemptCounts` state dropped:** The spec included `attemptCounts: Record<string, number>` in `AnsweringState`. Not needed — `attemptsRemaining` is carried inside each `AnswerResult` returned by the API.
- **`(authenticated)` route group not used:** Files placed at `src/app/assessments/[id]/` directly. Auth enforced per-page via `getUser()` as established in prior features.
