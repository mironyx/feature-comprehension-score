## Decisions made

- **No LLD** for this issue — it is a targeted bug fix with clear behaviour; the engine function is simple enough not to warrant a formal design doc.
- **`renderToStaticMarkup`** used for page-level HTML assertions in Vitest (Node.js environment); appropriate since server components can be awaited directly and no hydration is needed.
- **Thenable mock** (`Object.assign(Promise.resolve(...), {...chainMethods})`) for `assessment_participants` — required because the page awaits the query result directly (no `.order()`) but also chains `.eq().maybeSingle()` for the participant-lookup path.
- **No admin bypass** — Org Admins follow the same reveal rule as participants; this was an explicit product decision, tested in a dedicated gate case.
- **Rebase before merge** added to `/feature-end` — with multiple parallel agents, the base branch may have advanced; rebase + `--force-with-lease` ensures CI validates the integrated code.
