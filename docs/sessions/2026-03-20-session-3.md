## Decisions made

### CRLF/LF mismatch required a reconciliation migration

`supabase db diff` compares function bodies byte-for-byte between two shadow databases. Migration `20260317000002_get_effective_config_context_patterns.sql` had CRLF line endings; the new schema files use LF. This caused `get_effective_config` to appear in the diff with an apparently identical body. Root cause identified by querying `pg_proc.prosrc` directly via Docker and observing `\r` characters. Fix: convert schema files to LF, generate a reconciliation migration via `db diff -f`, apply via `db reset`.

### `.gitattributes` scoped to SQL only for now

Extending `.gitattributes` to all text files (`* text=auto eol=lf`) would require a `git add --renormalize .` normalisation commit touching all tracked files — a large whitespace-only diff. Deferred to issue #76 to keep PR #75 focused.
