## Decisions made

- **Standalone migration rather than combined with Story 2.1.** The LLD and issue body
  planned a single migration bundling E1 (hint column) and E2 (comprehension depth) schema
  changes. Since #215 (Story 2.1) hadn't started, the team lead directed to ship a
  standalone migration and let Story 2.1 add its own on top. Noted transparently in the PR
  body and captured as an implementation note in the LLD.
- **Excluded `package-lock.json` from the commit.** The only change was `fsevents` gaining
  a `"dev": true` flag — unrelated environment noise. Kept the commit surgical.
- **Five BDD specs instead of the three the LLD listed.** The exhaustive feature-stage
  tests memory captures this preference: added explicit-null and per-question independence
  cases, and tightened "backward compat" to verify all pre-existing column values plus the
  `awaiting_responses` status transition.
