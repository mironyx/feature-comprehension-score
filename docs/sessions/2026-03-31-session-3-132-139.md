## Decisions made

- **Reuse `fcs/service.ts` exports rather than duplicate logic.** Initial implementation duplicated `fetchRepoInfo`, branded ID types, and `triggerRubricGeneration` in a new standalone service file (~120 lines). The DRY violation was caught post-merge and required an additional refactor commit. The auth gap and silent-swallow were caught by PR review.

### Improvement actions

Three concrete fixes — tracked for retro:

1. **Add silent-swallow grep to `/feature` pre-PR step** — before raising a PR, run:
   ```bash
   grep -rn "catch" src/ --include="*.ts" | grep -v "// fire-and-forget\|logger\.\|console\.\|setError\|throw"
   ```
   Any unguarded catch blocks must be resolved before the PR is created. Automated — no agent discipline required.

2. **Add auth helper grep to CLAUDE.md API route section** — "Before implementing a new endpoint, grep `src/app/api/` for existing auth and authorisation helpers and use them." Now added to Coding Principles as the general **Read before writing** rule.

3. **Require auth helper reference in API route issue bodies** — add an "Auth" field to issue bodies for API routes naming the helper to use. Closes the design contract gap where `403 for non-admin` was specified but `assertOrgAdmin` was not mentioned.
