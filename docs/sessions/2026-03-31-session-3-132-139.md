## Decisions made

- **Reuse fcs/service.ts exports rather than duplicate logic.** The retry service was simplified
  to a thin adapter that delegates to `assertOrgAdmin` and `retriggerRubricForAssessment` from
  `fcs/service.ts`. This avoids duplicating `fetchRepoInfo`, branded ID types, and
  `triggerRubricGeneration`. LLD §2d.1 originally specified standalone service; updated via
  lld-sync.

- **Fire-and-forget pattern preserved.** The retry endpoint returns 200 immediately while rubric
  generation runs asynchronously, matching the existing `createFcs` pattern. If generation fails
  again, status returns to `rubric_failed` for another retry attempt.
