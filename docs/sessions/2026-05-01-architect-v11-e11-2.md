## Decisions

- **Decomposition (6 task issues).** Epic spans schema + new API endpoint + URL migration + new-form page + dashboard list + pending-queue rewrite. Total PR diff well above the 200-line budget. File-disjoint seams allowed clean splits along DB / API / route-move / FE pages.
- **No new ADRs.** ADR-0027 (project-as-sub-tenant) and ADR-0029 (Repo-Admin snapshot) already cover the cross-cutting decisions. Story-local choices (per-repo gate location, `ON DELETE SET NULL` for `assessments.project_id`, route-mismatch 404 at page layer not API layer) live in the LLD.
- **`assessments.project_id` is nullable + CHECK-gated for FCS.** Clean way to keep PRCC rows nullable (HLD requirement) without schema branching. CHECK uses `(type <> 'fcs' OR project_id IS NOT NULL)`.
- **`ON DELETE SET NULL` (not CASCADE).** Story 1.5 already prevents deleting a project with assessments via the empty-only DELETE; SET NULL is the safer fallback if that path is bypassed. Audit data preserved.
- **Legacy `/api/fcs` removed, not redirected.** Pre-prod, no callers in the wild. Reusable rubric pipeline helpers relocated to `src/lib/engine/fcs-pipeline.ts` so retry-rubric and the new endpoint share them.
- **404 enforcement on pid/aid mismatch happens at the page layer, not the API.** Minimises API blast radius for T2.3 and avoids breaking direct API calls with new query params.
- **`/assessments` rewrite drops the Completed tab.** V11 navigation model spec'd My Pending Assessments only; completed FCS rows are now reachable from the project dashboard list (T2.5).
