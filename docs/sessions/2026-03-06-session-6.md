## Decisions made

- **"PR context" replaces "artefacts"** — Clearer terminology for the data extracted from PRs (diff, files, description, linked issues, tests) and fed to the LLM for question generation.
- **FCS participants auto-suggested from PRs** — Rather than manual username entry (Story 3.1), the system auto-suggests participants from the authors and reviewers of the selected merged PRs. Org Admin can add/remove before confirming. Requires a requirements update to Story 3.1.
- **One LLM call per answer rationale documented** — Each answer scored in isolation to prevent cross-contamination between participants' answers (Story 4.2).
- **PRCC-only transitions explained** — Non-trivial/trivial commit handling and gate skip apply only to PRCC because FCS operates on merged PRs (no new commits, no merge gates).
- **Check Run completion count updated on reviewer changes** — Not just on participant submission.
