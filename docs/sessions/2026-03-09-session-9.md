## Decisions made

- **`--ignore-health-check` on WSL** — analytics requires Docker TCP (`localhost:2375`) which is not enabled by default in Docker Desktop on WSL. Disabling analytics and storage in `config.toml` is the right long-term fix; `--ignore-health-check` is the runtime workaround until Docker Desktop is configured or upgraded.
- **Hand-authored `types.ts`** — generated types require a running local instance. Hand-authoring unblocks development; run `npx supabase gen types typescript --local` after `supabase start` to keep in sync with schema changes.
- **`.env.test.local`** (gitignored) — holds actual JWT keys for the local instance. Defaults in `supabase-env.ts` match CLI v2.77 output so the file is optional.
