## Decisions Made

### OpenAI SDK signal threading (Fix D)
The OpenAI SDK accepts `signal` as a second-argument request option, not in the body:
```typescript
this.client.chat.completions.create(
  { ...args, signal: undefined } as never,
  { signal: args.signal },
)
```
This required adding `signal?: AbortSignal` to `SdkRequest` in `tool-loop.ts`. The `as never` cast strips the signal from the body object because the SDK body type doesn't declare it.

### `currentInterval(pollCount)` — explicit parameter
The LLD's pseudocode showed `currentInterval()` closing over `pollCount`. Implementation passes `pollCount` explicitly for clarity and testability. Documented in the updated LLD.

### Adaptive polling timing (21 fast intervals)
The initial `setTimeout` fires before `pollCount` increments, so polls 1–21 all fire at FAST_INTERVAL (not 20). Total = 21×3s + 24×10s = 303s ≥ 300s. The LLD timing comment was corrected post-implementation.

### isAdmin threading through create-assessment-form (Fix B scope gap)
The LLD's Fix B only named `page.tsx`, `polling-status-badge.tsx`, and `poll-status.ts`. The PR review caught that `create-assessment-form.tsx` hardcoded `admin={true}` on `PollingStatusBadge` — violating defence-in-depth even though the page redirects non-admins. Fixed by threading `isAdmin` as a prop from `new/page.tsx` → `CreateAssessmentForm` → `CreationProgress` → `PollingStatusBadge`. LLD updated to record the scope gap.

### Feature evaluator found 5xx test gap
The LLD's I1 says "429, 5xx" but test-author sub-agents only wrote 429 tests. Evaluator wrote 2 adversarial tests for 500 and 503. Both passed — the implementation was correct; only the test coverage was missing.
