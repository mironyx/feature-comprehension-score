## Decisions Made

- **`route-handler-readonly.ts`** — new file in `src/lib/supabase/`. The auth functions only accept
  `request`, so a client with a no-op `setAll` is needed. This is distinct from `route-handler.ts`
  which requires both request and response and writes session refresh cookies back.

- **Error field checking** — `extractUser` and `requireOrgAdmin` both check the Supabase `error`
  field and throw `ApiError(500)` on infrastructure failures. PR review caught that the spec was
  silent on this, which would have masked network/DB errors as 401/403.

- **`console.error` in `handleApiError`** — unknown errors are logged before returning 500.
  Review finding: silent discard violates observability principle.

- **`ZodType<T>` not `ZodSchema<T>`** — `ZodSchema` is deprecated in Zod v4 (^4.3.6 in use).

- **`validateParams()` deferred** — listed in the spec but not required by any current issue.
