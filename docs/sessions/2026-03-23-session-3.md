## Decisions Made

- **Vault over pgsodium** — pgsodium is inaccessible to `postgres` role on Supabase cloud.
  Vault is the supported abstraction; it uses pgsodium internally but exposes functions the
  `postgres` role can call.
- **Hand-authored migration** — `supabase db diff` cannot run when any prior migration
  references an unavailable extension. Documented in migration header and LLD §2.1.
- **Existing tokens deleted on migration** — tokens encrypted with pgsodium are unrecoverable
  without the key. Users re-authenticate to refresh. Acceptable: no real users exist yet.
- **`vault.update_secret` for rotation** — on re-authentication the existing secret UUID is
  retained and the secret rotated in-place, keeping `user_github_tokens` stable.

## Review Feedback Addressed

PR review (two parallel agents) caught one blocker:

- **`s.secret` → `s.decrypted_secret` in `get_github_token`** — `vault.decrypted_secrets.secret`
  holds the encrypted ciphertext; `decrypted_secret` is the AES-decrypted plaintext. Fixed in both
  migration and schema file. LLD updated to name the column explicitly.
