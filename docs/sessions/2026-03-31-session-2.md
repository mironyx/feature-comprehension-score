## Decisions made

- **No MSW, use `page.route()` + direct DB seeding:** Simpler than MSW for E2E since
  we only need to intercept one browser POST. Server-side calls are avoided by using
  email/password auth (no GitHub OAuth → no `link_participant` RPC).
- **Separate E2E helpers (no `@/` imports):** Playwright runs outside Next.js, so
  `@/` path aliases don't resolve. Created standalone `e2e-auth.ts` and `e2e-seed.ts`
  rather than refactoring existing helpers.
- **`test.describe.serial`:** Tests share mutable DB state (participant status changes
  between submit and results tests), so parallel execution would cause race conditions.
- **`postbuild` script for static assets:** Next.js `output: 'standalone'` doesn't
  copy `.next/static/` into the standalone output. Without client JS, React never
  hydrates — forms render as inert HTML but event handlers never attach.

## Key debugging insight

The submit button staying disabled was NOT an input method issue (`fill()` vs `type()`
vs `pressSequentially()`). The root cause was that `.next/static/` was missing from the
standalone server output, so **React never hydrated**. Native HTML textareas accepted
keyboard input (values visible in page snapshot), but React event handlers were never
attached, so `onChange` never fired and component state never updated.

### Improvement actions

- **Build verification first:** Always run `npm run build && npm run start` and
  manually verify the standalone server serves client JS before writing E2E tests.
  Would have caught the static assets issue immediately.
- **Check page snapshot for hydration:** When React forms don't respond to input,
  check whether `__reactFiber$` properties exist on DOM elements — if not, hydration
  failed. Don't cycle through input methods.
- **`.env.test.local` needs `NEXT_PUBLIC_*` vars:** Document that E2E builds must use
  local Supabase URLs because `NEXT_PUBLIC_*` vars are baked at build time (including
  middleware cookie name derivation).
