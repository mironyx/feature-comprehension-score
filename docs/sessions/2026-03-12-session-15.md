### TypeScript language server module resolution fix

**Root cause:** VS Code TS language server picks `tsconfig.json` (which has `"plugins": [{"name": "next"}]`) for test files. The Next.js plugin interferes with module resolution for non-Next.js files, causing `Cannot find module` errors even when files exist on disk and `tsc --noEmit` is clean.

**Fix:**

- **`tsconfig.json`** — excluded test files so the language server skips it for them:
  ```json
  "exclude": ["node_modules", "**/*.test.ts", "**/*.test.tsx", "tests/**/*"]
  ```
- **`tests/tsconfig.json`** (new) — language server uses nearest `tsconfig.json`; this one extends the base with `"plugins": []`, no Next.js plugin, inherits `@/*` path alias:
  ```json
  {
    "extends": "../tsconfig.json",
    "compilerOptions": { "plugins": [] },
    "include": ["**/*.ts"],
    "exclude": ["../../node_modules"]
  }
  ```

Errors resolved:
- `Cannot find module './client'` in `src/lib/engine/llm/client.test.ts`
- `Cannot find module '@/lib/supabase/types'` in `tests/helpers/supabase.integration.test.ts`
