## Decisions made

- **Inactive hover token = `hover:text-text-primary`, not `hover:text-accent`.** The pre-existing NavBar used `hover:text-accent`, but that violates the issue's BDD spec _"highlights only one link at a time"_ — every link would carry the substring `text-accent`. Switched to `hover:text-text-primary`, matching the § T8 inactive-tab pattern. Recorded as Implementation note in LLD § T2.
- **Dynamic-route breadcrumbs deferred (still).** § T1 marked `[id]` / `/results` / `/submitted` as _(deferred → issue #341)_, but #341's AC don't require them and implementing them needs server-side `feature_name` plumbed into the client `BreadcrumbsBar`. Marked as a follow-up; LLD § T1 updated accordingly.
- **`NavLink` type exported with `readonly` fields.** LLD spec showed it as private and mutable. Exported because `nav-bar.tsx` imports it; `readonly` because the link list is data, not state.
- **Single PR for both NavLinks + layout restructure.** The two changes are coupled — moving the assessment pages exposes the NavBar on those routes, which is what makes the active-route highlighting visible there. Splitting would have produced an intermediate state where NavBar shows but inactive styling still applied broadly.
