## Decisions made

- **Server-side context loading** — page fetches context via `loadOrgPromptContext` (same helper used by rubric generation) rather than a client-side GET call. Simpler, no loading state needed.
- **No LLD for this feature** — design reference is the UI Surface section of `v1-prompt-changes.md`. Straightforward UI over existing API. LLD sync skipped.
- **Null return from `loadContext`** — PR review flagged synthetic empty row as type-unsafe. Changed to return `null`, route handler maps to `{context: {}}` response.
- **Explicit singular/plural labels** — PR review flagged `label.slice(0, -1)` as fragile. Switched to explicit `singular` and `plural` parameters.

## Review feedback addressed

1. Duplicate React keys in TagInput — switched to `key={\`${item}-${i}\`}`
2. Synthetic `OrgContextRow` with empty strings — replaced with `null` return
3. Fragile `label.slice(0, -1)` — explicit singular/plural params
