## Original session (2026-03-23, session ID `0042356a`)

### What was built

Migrated `store_github_token` from pgsodium to Supabase Vault (#84). Root cause: pgsodium
crypto functions (`crypto_aead_det_encrypt`) are owned by the system superuser on Supabase
cloud; the `postgres` role cannot be granted execute permission. Vault functions are accessible
to `postgres` on both cloud and local.

**Files changed:**

- `supabase/schemas/tables.sql` — replaced `encrypted_token + key_id` with `token_secret_id uuid` on `user_github_tokens`
- `supabase/schemas/functions.sql` — rewrote `store_github_token` (Vault upsert pattern); added `get_github_token` reading `vault.decrypted_secrets`
- `supabase/migrations/20260323163850_vault_token_migration.sql` — hand-authored (shadow DB cannot apply pgsodium migration, making `db diff` impossible; documented in migration header and LLD §2.1)
- `supabase/migrations/20260323153000_pgsodium_extension_and_grants.sql` — wrapped in exception handlers so `db reset` succeeds without pgsodium installed locally
- `src/lib/supabase/types.ts` — updated `user_github_tokens` row type; added `get_github_token` RPC signature
- `docs/design/lld-phase-2-web-auth-db.md` — §2.2 rewritten to reflect Vault storage; §2.1 notes hand-authored migration exception
- `.claude/settings.json` — added WebSearch, WebFetch, post-merge cleanup commands to allow list
- `.claude/skills/feature-end/SKILL.md` — added Step 2.5 (final cost query)

### Key decisions

- **Vault over pgsodium:** Vault is the only secure option accessible to `postgres` on Supabase cloud.
- **Hand-authored migration:** Shadow DB limitation documented in migration header and LLD.
- **Existing tokens deleted:** Tokens encrypted with pgsodium are unrecoverable; users re-authenticate. Acceptable at this stage (no real users).
- **`vault.update_secret` for rotation:** Existing secret UUID is retained on re-auth; only the secret value rotates in-place.

### Review findings addressed

One blocker found by the parallel review agents:

- **`s.secret` → `s.decrypted_secret`:** `vault.decrypted_secrets.secret` is the encrypted ciphertext; `decrypted_secret` is the AES plaintext. Fixed in migration, schema file, and LLD.
