## Decisions made

- **JSONB parameter passing**: raw JS objects/arrays passed directly to `.rpc()`, not
  `JSON.stringify()`. PostgreSQL JSONB columns accept JS objects through the Supabase client.

  TypeScript type incompatibility between refined types and the generated `Json` type.
- **`CreateAssessmentParams` single-use struct**: accepted despite CLAUDE.md prohibition,
  justified by CodeScene "Excess Number of Function Arguments" threshold. Added justification
  comment.

  union literals (`'prcc' | 'fcs'`, `'active' | 'inactive'`, etc.).
