## Decisions made

- **Service role for all data queries:** RLS on `assessment_participants` limits users to their own row, making full participant counts impossible without the service role. Pattern is consistent with existing API routes.
- **Reference answers shown to all participants:** Issue #104 explicitly states "FCS is educational — show reference answers". This overrides Story 6.2 in `v1-design.md` which restricted reference answers to Org Admins only. Design doc flagged for future update.
- **No status guard on results page:** The issue does not require that scoring be complete before viewing — the `scoring_incomplete` notice handles partially-scored states. Avoids coupling the page to assessment lifecycle states.
