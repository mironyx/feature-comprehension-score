## Decisions made

- **Application-layer `updated_at`** over trigger — simpler, and all writes go through our API/webhook handler.
- **`github_role` as single source** — no redundant `is_admin` boolean. RLS function maps GitHub roles to admin status.
- **RESTful API** — resource-oriented endpoints instead of RPC-style action endpoints. Monitoring handled via structured application logging rather than URL-based differentiation.
- **Keep `superseded_by`** on assessments — explicit chain linking for invalidated → replacement assessments (traces to Story 2.8).
- **Keep `'participant'` contextual role** — needed for FCS where people aren't distinctly author/reviewer on the feature-level assessment.
