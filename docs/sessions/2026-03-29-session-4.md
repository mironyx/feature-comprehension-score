## Decisions made

- **`link_participant` concurrency:** The RPC runs in `Promise.all` alongside `fetchAssessment` (not sequentially before it) because it has no dependency on assessment data. `fetchParticipant` runs after both settle. This saves one Supabase round-trip on every first-visit page load.
- **Link failures are best-effort:** If `link_participant` fails, the error is logged but execution continues. The participant sees "Access Denied" on that visit; a refresh after the transient error clears will recover. Documented in a code comment.
- **`parseInt` vs `Number`:** `user_metadata.provider_id` is parsed with `parseInt(raw, 10)`. The Supabase-generated type for `p_github_user_id` is `number`, so this matches the type contract. Current GitHub IDs are well within `Number.MAX_SAFE_INTEGER`.
