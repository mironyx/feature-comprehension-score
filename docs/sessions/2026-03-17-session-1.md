## Decisions made

- `org_config.context_file_patterns` is `NOT NULL DEFAULT '{}'` — always has a value,
  defaults to no patterns
- `repository_config.context_file_patterns` is nullable — null means inherit from org
- `get_effective_config` uses `COALESCE(rc.context_file_patterns, oc.context_file_patterns)`
  consistent with all other config fields
- RLS not changed — existing policies cover all columns at row level; no column-level
  policy needed

## Learnings / process improvements

- **Branch workflow:** `feat/assessment-engine` served as integration branch in early sessions
  but `main` is now the correct base. Update `CLAUDE.md` to reflect this.
- **Migration generation:** Supabase has built-in declarative schema support
  (`supabase/schemas/` + `supabase db diff`). Worth adopting for future migrations — define
  desired state in schema files, generate migrations automatically. See:
  <https://supabase.com/features/declarative-schemas>
- **Token/timing metrics:** Not available via hooks in subscription mode. OpenTelemetry
  (`CLAUDE_CODE_ENABLE_TELEMETRY=1`) is the correct mechanism for per-session token
  aggregation.
- **PR rebasing:** Always `git fetch origin && git pull` on `main` before rebasing a feature
  branch — local and remote main can diverge if others push in parallel.
