## Decisions made

- **Role-based rendering via `caller_role` field** — exposing `caller_role` in the API response avoids a second auth check in the page, keeps the page thin, and allows the page to branch cleanly on a single field.
- **No new design system tokens** — all V8 UI work uses existing Tailwind tokens and components from `frontend-system.md` v0.1. No amendments to the design system doc needed.
- **T2 shares files with T1 (route.ts, service.ts) in Epic 2** — encoded as a hard sequential dependency, not a soft coupling.
- **No pagination for repository list** — requirements explicitly cap expected repo count at < 100 per org.
- **Server-side GitHub API call for accessible repos** — installation token never reaches the client, as required by the security constraints in the requirements doc.
