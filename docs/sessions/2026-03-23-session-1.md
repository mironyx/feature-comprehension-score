Removed hardcoded Supabase CLI default JWT tokens from `tests/helpers/supabase-env.ts`. These were the well-known demo JWTs committed by the Supabase CLI scaffold. Replaced with empty-string defaults and a comment directing developers to copy keys from `supabase start` output into `.env.test.local`.

### E2E fix

`playwright.config.ts` — added `webServer.env` block to pass `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`, and `SUPABASE_SECRET_KEY` explicitly to both `npm run build` and the standalone server. Needed because Next.js middleware runs in the Edge Runtime and does not inherit `.env.local` from the filesystem at server start time.

## Decisions Made

- **Empty-string fallback for integration test keys**: intentional. The old hardcoded JWTs were a security risk (secrets in source). The new behaviour requires `.env.test.local` for integration tests, which is documented in the comment block.
- **No SDK changes**: Supabase JS SDK v2 accepts both old and new key formats. This is a pure rename with no functional changes.
- **Playwright `webServer.env`**: preferred over modifying `npm run start` script, to keep the start command simple and avoid `cross-env` dependency for a one-off env block.
