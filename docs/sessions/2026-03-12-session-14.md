# Session 14 — 2026-03-12

## Focus

Set up VS Code diagnostics pipeline for Claude Code — automatic code quality feedback from CodeScene and other VS Code extensions.

## Completed

- **Diagnostics pipeline architecture** — Designed and implemented a two-channel system for consuming VS Code extension diagnostics:
  1. **PostToolUse hook** (`.claude/hooks/check-diagnostics.sh`) — automatically runs after Write/Edit on source files, waits 3s for the diagnostics-exporter extension, injects issues as inline context
  2. **`/diag` skill** (`.claude/skills/diag/SKILL.md`) — manual batch check across all changed files
- **Agent definition** (`.claude/agents/diagnostics-checker.md`) — documents the JSON format for reference
- **CLAUDE.md updated** — new "Code Quality — Diagnostics Pipeline" section replacing the old CodeScene-specific section
- **`.gitignore` updated** — added `/.diagnostics` to exclude exported diagnostics from git
- **End-to-end test** — verified the full pipeline:
  - Introduced intentional TS errors (`let=1;`, `const x: number = "not a number"`)
  - Extension exported diagnostics to `.diagnostics/tests/helpers/supabase.integration.test.ts.json`
  - Background agent correctly reported errors
  - `/diag` skill correctly reported errors
  - **PostToolUse hook fired successfully** after Edit, injecting diagnostics inline
- **Hook debugged** — initial version used `jq` (not installed on Windows), rewrote to use Node.js for JSON parsing; fixed stdin reading for Windows path handling

## Decisions

- **Hook + skill (not sub-agent only)**: Originally planned a background sub-agent approach, but a PostToolUse hook is simpler and fully automatic. The skill provides manual batch checking. Both channels complement each other.
- **Node.js over jq for hooks**: `jq` not available on Windows; Node.js is always present in this project. Hook script uses inline Node.js via `node -e`.
- **Custom agents can't be used as `subagent_type`**: Discovered that `.claude/agents/*.md` definitions aren't available via `Agent(subagent_type=...)`. They serve as reference documentation. Use general-purpose agents with inline prompts instead.
- **Short command names**: User preference for `/diag` over `/check-diagnostics`.

## Diagnostics JSON format

The diagnostics-exporter extension writes files at `.diagnostics/<relative-path>.json`:

```json
{
  "file": "relative/path/to/source.ts",
  "analyzedAt": "2026-03-12T11:53:41.861Z",
  "diagnostics": [
    { "source": "ts", "severity": "Error", "message": "...", "line": 42, "column": 5, "code": 1214 }
  ]
}
```
